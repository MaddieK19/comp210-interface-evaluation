Heuristics Analysis - Qualitative

The first evaluation method is a heuristics analysis. This is a qualitative method that looks at whether Skyrim meets all the chosen heuristics. The heuristics used are Pinelle's heuristics for video games. These heuristics were specifically designed for game usability. Pinelle defines "game usability as the degree to which a player is able to learn, control, and understand a game”. The method used is Nielsen’s. Nielsen says that a heuristics analysis should be carried out by a "small set of evaluators" to maximise the number of issues found however due to time constraints I carried out the analysis myself. This means that some issues will be missed.

A heuristics analysis will be conducted on both Skyrim and the SkyUI mod.

Provide consistent responses to the user’s actions.

In Skyrim the consistency of the responses often depends on the user and their input device. An Xbox controller will likely give consistent results however there are some issues when using a mouse and keyboard. For example, switching between mouse and keyboard in the menu or inventory screens can cause strange results where the wrong option is selected because the mouse was resting on one option and the arrows being used to get another. A way to improve this would be to only allow one method such as the mouse to be used in the menu screens, this would prevent any inconsistent results. The input devices are the same for both Skyrim and SkyUI so both provide similar responses.


Allow users to customize video and audio settings, difficulty and game speed

On PC Skyrim auto-detects the ideal video and audio settings for the player’s computer. There is also the option to manually change any of those settings, however changing them above what the computer can run will affect how the game runs potentially making it unplayable. Audio and video settings can only be changed before the game is launched using a separate tool or once them is running from the pause menu. The image below shows the pause menu where the player can access sub menus for settings for audio, visual and game play.

The combination of both auto detect and customising the controls allows players to just start the game without altering settings or allow other players to customise their settings.

Provide unobstructed views that are appropriate for the user’s current actions and provides users with information on game status.

The constant gameplay UI in Skyrim is minimal as it’s often just a compass at the top of the screen shows nearby landmarks, currently marked quests and nearby enemies at all times. The settings menu allows the user to alter the transparency of the compass if it is obstructing their views. SkyUI does not alter this part of the UI, however there are alternative mods that let the player toggle the compass UI on and off such as iHUD. On the bottom half of the screen there are three bars representing magicka, health and stamina levels. These bars only appear when in combat or when one is changing in value for example the player is taking damage or casting a spell. These three bars make it clear to the player if either three of those values are being effected. However, if the player is being affected by something such as poison or a spell it isn't always clear the player has to check in a menu to see what effects are active on them. SkyUI uses the same three bars but also any effects active on the player are shown in the top right corner, SkyUI presents this information on screen to player making it much more accessible as the player does not have to navigate through a couple of menu screens to see the information. This is a change that could be recommended for the original game as this information could be vital in combat.

Provide intuitive and customizable input mappings and provides controls that are easy to manage, and that have an appropriate level of sensitivity and responsiveness.

The keyboard controls are easily changed in an in game menu for controls, there's also a game play menu that allows for axis inverting and mouse sensitivity altering. Both these features imrpove the usability of the game allowing players to tailor the input settings to their play style.

Provide visual representations that are easy to interpret and that minimize the need for micromanagement.

Allow users to skip non-playable and frequently repeated content.

Provide instructions, training, and help.

The game starts for a tutorial teaching players the basic controls however some in game systems aren't explained and have to be learned through playing the game.
