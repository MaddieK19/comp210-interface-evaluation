Heuristics Analysis - Qualitative

The first evaluation method is a heuristics analysis. This is a qualitative method that looks at whether Skyrim meets the chosen heuristics. The heuristics used are Pinelle's heuristics for video games. These heuristics were specifically designed for game usability. Pinelle defines "game usability as the degree to which a player is able to learn, control, and understand a game”. The method used is Nielsen’s. Nielsen says that a heuristics analysis should be carried out by a "small set of evaluators" to maximise the number of issues found however due to time constraints I carried out the analysis myself. This means that some issues will be missed.

A heuristics analysis will be conducted on both Skyrim and the SkyUI modification to compare the two interfaces and see if SkyUI does imrpove the Skyrim interface.

Provide consistent responses to the user’s actions.

In Skyrim the consistency of the responses often depends on the user and their input device. An Xbox controller will likely give consistent results however there are some issues when using a mouse and keyboard. For example, switching between mouse and keyboard in the menu or inventory screens can cause strange results where the wrong option is selected because the mouse was resting on one option and the arrows being used to get another. A way to improve this would be to only allow one method such as the mouse to be used in the menu screens, this would prevent any inconsistent results. The base mechanics such as hit detection and movement all appear to operate appropriately however there can be physics issues with objects flying away at greater speeds than they should. The input devices are the same for both Skyrim and SkyUI so both provide similar responses.


Allow users to customize video and audio settings, difficulty and game speed

On PC Skyrim auto-detects the ideal video and audio settings for the player’s computer. There is also the option to manually change any of those settings, however changing them above what the computer can run will affect how the game runs potentially making it unplayable. Audio and video settings can only be changed before the game is launched using a separate tool or once them is running from the pause menu. The image below shows the pause menu where the player can access sub menus for settings for audio, visual and game play.

There's also a game play menu where the player can change the difficulty at any time. They can also change when the game saves, how sensitive controls are and invert the Y axis to suit their play style.

Provide unobstructed views that are appropriate for the user’s current actions and provides users with information on game status.

The user can alter their view of the game using mouse controls to change angles or they can switch between first and third person. The constant gameplay UI in Skyrim is minimal as it’s often just a compass at the top of the screen shows nearby landmarks, currently marked quests and nearby enemies at all times. The settings menu allows the user to alter the transparency of the compass if it is obstructing their views. SkyUI does not alter this part of the UI, however there are alternative modifications that let the player toggle the compass UI on and off such as iHUD. On the bottom half of the screen there are three bars representing magicka, health and stamina levels. These bars only appear when in combat or when one is changing in value for example the player is taking damage or casting a spell. These three bars make it clear to the player if either three of those values are being effected. However, if the player is being affected by something such as poison or a spell it isn't always clear the player has to check in a menu to see what effects are active on them. SkyUI uses the same three bars but also any effects active on the player are shown in the top right corner, SkyUI presents this information on screen to player making it much more accessible as the player does not have to navigate through a couple of menu screens to see the information. This is a change that could be recommended for the original game as this information could be vital in combat.

Provide intuitive and customizable input mappings and provides controls that are easy to manage, and that have an appropriate level of sensitivity and responsiveness.

The keyboard controls are easily changed in an in game menu for controls, there's also a game play menu that allows for axis inverting and mouse sensitivity altering. Both these features improve the usability of the game allowing players to tailor the input settings to their play style. SkyUI does not affect the input mapping or controls.

Allow users to skip non-playable and frequently repeated content.

The beginning tutorial content cannot be skipped without the use of modifications. However, this content is not frequently repeated unless the player restarts the game. Players can skip conversations however some content may be repeated depending on how often the player saves. SkyUI has no effect on non-playable or repeated content.

Provide instructions, training, and help.

The game starts with a tutorial that teaches the player the basic controls. SkyUI does not add any more tutorials or training to the game.


Analysis

There are many areas of Skyrim that SkyUI does not affect however the results suggest that the areas it does affect it does improve on. It adds information to UI preventing the player from having to navigate through menus to find it and alters existing crafting and inventory UIs to make them easier to navigate. However, some of the changes may not be suited to all players, for example the addition of symbols to show active effects may not be clear making it easier for the player to use the original Skyrim UI to navigate to the information through the menus. The goal of SkyUI is to make the inventory system more "convenient for the player" and have more "intuitive and efficient keyboard navigation”, the results of the heuristics analysis suggest that SkyUI does improve on the pre-existing Skyrim interface. However, as this analysis was conducted by myself it is bias towards what I find easier to use in a UI. In the future to improve the reliability of the results I would get multiple people, ideally experts, to carry out the analysis and then compare the results.
